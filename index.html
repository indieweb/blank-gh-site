Welcome to Web Dev Guy

Hi, my name is Gabe Corsini and I am a human on the Internet. I’m working to become a primarily self‐taught Full Stack Web Developer, with the hopes of one day moving on from my current career and into the private sector Web Development space.

A little bit about me.

Currently, I am an active duty servicemember in the US Military. I have been in the service since 2012 and work as an Information Systems Technician, which is just the fancy way of saying I'm a military IT guy. In recent months, I have began reaching the conclusion that I am ready to exit the service and pursue something different. I've always had a passion for software development and programming, but struggled to learn many full programming languages (e.g. Java, C+, C#, etc) since I do not have a traditional college education. Not long ago, I fell into the web development space at the recommendation of my twin brother and I've been really enjoying it. So here we are, on this journey to become a Full Stack Web Developer so that, by the end of 2022 when it's time to exit the service, I have the experience and the know how to go find a job that I want and do what I love.

Here are my current goals, in order, for what I want to learn on my Web Development Journey. As I complete items, I will mark the boxes and supply a date at the end of the line for when I believe I accomplished the goal. This is subjective of course.

  • Learn more than just basic HTML5, as well as best formatting practices.
  • Get pretty good with CSS because it's cool.
  • Start learning Vanilla JS so I can make interactive pages.
  • Get a LOT better with Git because my first attempts were a shit show (Just don't read my commits for a while).
  • Use 1-3 to build a nice looking website.
  • Once nice looking website has been made, start learning React.js (Don't ask how I picked the framework because you'll be disappointed).
  • Build an even cooler website.
  • Re-learn about how the internet works (HTTP(S), DNS, GET/POST stuff, OSI vs TCP/IP models, etc).
  • Start learning Node.js for backend because I like the way it looks.
  • Make an even more cool website than the last two becuase it'll do backend things (I hope).
Skillset Explanation
HTML 5 I mean, it's the skeleton of the internet. Can't have a body without bones right? IMHO, a strong grasp of the basics allows for a firm foundation on which to build the rest.
CSS 3 The first step to a good looking and interactive webpage.I've been told that a week knowledge in this area can cause a lot of issues in an enterprise. I want my sites to look good and CSS has so many fun frameworks I want to try.
Javascript This is where interactivity really takes off. Plus, Vanilla Javascript is the foundation to all of the front and backend frameworks I want to try later one (Like Node, React, and eventually Vue and more)
React I honestly picked it because I like the way the logo looks and it's one of the most popular frameworks in the sector right now.
Git Version Control is everything these days, or so it seems. I'm not quite sure how the enterprise uses it yet but I want to be good at it.
Networking Concepts Once I start working on the backend, I want to dive back into my networking basics. HTTP, TCP/UDP, the TCP/IP model, DNS, and so on
Node One of the most popular backend frameworks available and competes with things like Python, Java, and C#. It may not expand as well as the others, but I think it's a good place to start for me at least.
